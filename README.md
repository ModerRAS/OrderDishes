# OrderDishes
捋需求

## 界面设计
1. 基本的用户管理
    1. 手动设定物业帐号密码
	1. 手动设定管理员帐号密码
	1. 每个人的帐号需要自行注册，使用邀请码方式注册
1. 点菜界面
    1. 有一个日期选择框
	1. 有一个早中晚饭的分界线
	1. 分界线中间有一些卡片，卡片里有图片有文字，可以点选
	1. 一个简化点菜界面，选一段日期，设定为这几天三顿饭全选
1. 订单界面，用来打饭
1. 做菜界面，用来看这顿饭每个菜要做多少
1. 提供每周菜谱界面
	1. 上传菜谱按键
	1. 下载菜谱按键，提供一个菜谱模板，将本周菜谱填入后上传。
1. 维护菜品
    1. 一个列表，列表中有每一个传过菜谱的菜名，对应菜名的图片，和替换图片的功能
1. 统计界面
    1. 统计每顿饭每个菜的选择量
	1. 统计每日流水等

## 数据库设计
使用Redis作为数据库，并且只用其中一个数据库
要做以下键值对
```
Guid -> 单个食物(Food)
食堂名称-单个食物名称 -> Guid
Guid -> 菜单(Menu)
食堂名称-日期-早中晚饭 -> Guid
Guid -> 用户(User)
用户名 -> Guid
Guid -> 订单(Order)
食堂名称-日期-用户-早中晚饭 -> Guid
```
